# Lightning Indexer Prolog Quant — PyPTO v3 Port

Self-contained port of `lightning_indexer_prolog_quant.py` to PyPTO v3.

## Layout

- **golden.py**: Single entry; `@pl.program` with InCore and Orchestration. No `tensor_functions` or `models`.
- **build_output/**: Generated by compile: `orchestration/*.cpp`, `kernels/aiv/*.pto`, `kernels/aic/*.pto`, `report/*.txt`.

## Compile

From the **pypto** repo root (e.g. `../../pypto`):

```bash
cd /path/to/pypto
export PYTHONPATH="$PWD:/path/to/pypto_src"
python -c "
from pypto import ir
from pypto.backend import BackendType
import golden
out = ir.compile(
    golden.LightningIndexerPrologQuantProgram,
    output_dir='/path/to/pypto_src/build_output',
    strategy=ir.OptimizationStrategy.PTOAS,
    dump_passes=False,
    backend_type=BackendType.PTO,
    skip_ptoas=True,
)
print('Output:', out)
"
```

Use **PTOAS** strategy (no InsertSync) and **BackendType.PTO**; optionally `skip_ptoas=True` to only emit `.pto` MLIR.

## Performance tuning (TILE + batch + sequence-length tiering)

- **Larger TILE** (`TM=TN=TK=64`): Increases SRAM use and kernel work per launch (see `report/memory_after_AllocateMemoryAddr.txt`; Vec ~30–42%, Acc ~12.5%).
- **Batch inside incore** (`BATCH_TILES=2`): Each incore processes two tiles (unrolled; PTO requires ConstInt load/store offsets) for longer kernel execution.
- **Sequence-length tiering** (choose at build/runtime):
  - **Short seq**: `TM=TN=TK=32`, `BATCH_TILES=4` — smaller tile, more tiles per kernel.
  - **Medium seq** (default): `TM=TN=TK=64`, `BATCH_TILES=2`.
  - **Long seq**: `TM=TN=TK=64`, `BATCH_TILES=1` — larger tile, single tile per kernel.

## Algorithm (from original)

- **Q path**: x → matmul(w_qb.T) → dequant(scale) → RoPE(cos, sin) → Hadamard(hadamard_q) → symmetric_quant → q_int8_out, q_scale_out
- **K path**: x → matmul(wk.T) → LayerNorm → RoPE → Hadamard(hadamard_k) → symmetric_quant → k_int8_out, k_scale_out
- **Weights path**: x → matmul(w_proj.T) → softmax → weights_out

All steps are expressed as InCore kernels (load → PTO-ISA tile ops → store) and one Orchestration function that loops over tiles and calls them.
